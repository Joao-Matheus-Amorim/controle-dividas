# Termo de Abertura - FamilyFinance

## Nome do projeto

FamilyFinance - Aplicativo financeiro familiar personalizado.

## Tipo de projeto

Projeto personalizado para uma unica familia, com app nativo Android/iOS planejado e painel web administrativo.

## Justificativa

A familia precisa centralizar o controle de gastos, limites mensais, contas a pagar, contas a receber, bancos, rendas e relatorios. O Admin familiar tambem precisa controlar o que cada usuario pode ver, criar, editar ou excluir.

## Objetivo geral

Construir uma solucao financeira familiar sob medida, com backend seguro, painel web administrativo e evolucao para aplicativo nativo Android e iOS.

## Objetivos especificos

- Controlar membros financeiros da familia.
- Definir limites mensais por pessoa.
- Registrar gastos por pessoa e categoria.
- Controlar contas a pagar.
- Controlar contas a receber.
- Gerenciar bancos por pessoa.
- Gerar relatorios financeiros.
- Permitir Admin familiar com controle de permissoes.
- Preparar a evolucao para app nativo.

## Patrocinador

Responsavel financeiro da familia.

## Gerente do projeto

Joao Matheus.

## Premissas

- O app sera personalizado para uma familia especifica.
- O projeto nao sera vendido como SaaS nesta fase.
- O backend sera Supabase.
- O painel web atual sera mantido como backoffice.
- O produto final desejado sera mobile-first.
- O Admin familiar tera controle de permissoes.

## Restricoes

- Evitar expansao para multiplas familias nesta fase.
- Evitar escopo comercial ou assinatura.
- Priorizar simplicidade de uso familiar.
- Validar MVP antes de implementar app mobile completo.

## Riscos iniciais

- Escopo crescer para produto comercial antes da validacao familiar.
- Permissoes ficarem complexas demais.
- Diferenca entre usuario do sistema e membro financeiro gerar confusao.
- App mobile ser iniciado antes da documentacao aprovada.
- Custos externos de publicacao e build serem esquecidos.

## Criterios de sucesso

- Build web aprovado.
- Banco sem dados duplicados de seed.
- Fluxo financeiro principal funcionando.
- Documentacao alinhada ao objetivo familiar personalizado.
- Roadmap mobile aprovado antes da implementacao.
