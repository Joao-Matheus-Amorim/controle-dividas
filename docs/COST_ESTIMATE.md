# FamilyFinance - Estimativa de Custo

## Contexto

Esta estimativa considera o FamilyFinance como projeto personalizado para uma unica familia, com app nativo Android/iOS e painel web administrativo.

Nao considera, nesta fase, SaaS publico, multiplas familias, assinatura, area comercial ou venda em escala.

## Escopo de custo

A estimativa considera:

- documentacao e gestao do projeto;
- estabilizacao do MVP web;
- app nativo Android e iOS;
- Admin familiar;
- permissoes por modulo e acao;
- backend Supabase;
- publicacao inicial;
- validacao e testes manuais.

## Fase 1 - Documentacao e planejamento

Entregas:

- README profissional.
- Visao do produto.
- Estrategia mobile.
- Escopo personalizado.
- WBS/EAP.
- Requisitos.
- Riscos.
- Criterios de aceite.
- Parecer de custo.

Estimativa:

```txt
R$ 2.500 a R$ 6.000
```

## Fase 2 - Estabilizacao do MVP web

## Custos externos esperados

- Apple Developer Program: custo anual para publicacao na App Store.
- Google Play Console: taxa unica de cadastro.
- Expo/EAS: pode iniciar gratuito ou com plano pago conforme frequencia de builds.
- Supabase: pode iniciar gratuito e evoluir para plano pago em producao.
- Dominio e hospedagem web: conforme estrategia de deploy.

## Parecer

Para esta familia, a recomendacao e nao iniciar como produto comercial. O investimento deve ser faseado, com validacao do MVP antes da implementacao mobile completa.

## Estrategia comercial recomendada

Apresentar o projeto em fases:

1. Planejamento e documentacao.
2. MVP web estabilizado.
3. App mobile MVP.
4. Admin e permissoes.
5. Modulos avancados.

Assim o cliente entende valor, custo e evolucao sem misturar projeto familiar com produto SaaS.
